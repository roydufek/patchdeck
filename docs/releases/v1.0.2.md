# Patchdeck v1.0.2 (draft)

Release date: 2026-03-30

## Summary
This release improves bulk-action UX and correctness in host operations.

## Fixed
- Bulk apply progress could be confusing when selected hosts exceeded eligible hosts (e.g., `Applying 4/4` while 5 were selected).

## Improved
- Bulk apply now performs a post-apply scan on each host **before moving to the next host**.
- Host cards refresh progressively during bulk apply so package/restart status is current per-host.
- Confirmation dialog now shows eligibility context when selected > eligible.
- Progress indicator now shows eligibility context (`eligible of selected`) for clearer operator feedback.
- Completion toast includes eligibility context when not all selected hosts were eligible.

## Files changed
- `web/src/components/Dashboard.jsx`

## Notes for deployment
- Standard containerized flow: build/publish via GitHub → GHCR image pipeline.
- No API schema changes in this release.

## Validation checklist
- [ ] Bulk apply with all selected hosts eligible.
- [ ] Bulk apply where one or more selected hosts are ineligible.
- [ ] Confirm per-host post-apply scan updates cards before next host starts.
- [ ] Confirm completion toast reports `success/total` correctly.
